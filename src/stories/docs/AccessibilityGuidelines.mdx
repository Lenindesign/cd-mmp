  <div className="a11y-section">
    <h2>Standards</h2>
    <p>
      All components in this design system are built to meet WCAG 2.1 Level AA compliance. This ensures our interfaces are perceivable, operable, understandable, and robust for all users.
    </p>
    
    <table className="a11y-table">
      <thead>
        <tr>
          <th>Principle</th>
          <th>Requirement</th>
          <th>Our Approach</th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <td><strong>Perceivable</strong></td>
          <td>Content must be presentable in ways users can perceive</td>
          <td>Alt text, color contrast, text alternatives</td>
        </tr>
        <tr>
          <td><strong>Operable</strong></td>
          <td>Interface must be navigable and usable</td>
          <td>Keyboard navigation, focus management</td>
        </tr>
        <tr>
          <td><strong>Understandable</strong></td>
          <td>Information and UI must be understandable</td>
          <td>Clear labels, error messages, predictable behavior</td>
        </tr>
        <tr>
          <td><strong>Robust</strong></td>
          <td>Content must work with assistive technologies</td>
          <td>Semantic HTML, ARIA attributes</td>
        </tr>
      </tbody>
    </table>
  </div>

  <div className="a11y-section">
    <h2>Color Contrast</h2>
    <p>
      Text and interactive elements must have sufficient contrast against their backgrounds to be readable by users with low vision.
    </p>
    
    <table className="a11y-table">
      <thead>
        <tr>
          <th>Element</th>
          <th>Minimum Ratio</th>
          <th>Our Ratio</th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <td>Normal text (under 18px)</td>
          <td>4.5:1</td>
          <td>✓ 7:1+ (Dark Grey on White)</td>
        </tr>
        <tr>
          <td>Large text (18px+ bold or 24px+)</td>
          <td>3:1</td>
          <td>✓ 7:1+</td>
        </tr>
        <tr>
          <td>UI components and graphics</td>
          <td>3:1</td>
          <td>✓ 4.5:1+</td>
        </tr>
        <tr>
          <td>Focus indicators</td>
          <td>3:1</td>
          <td>✓ Uses brand blue (#1B5F8A)</td>
        </tr>
      </tbody>
    </table>

    <div className="a11y-do-dont">
      <div className="a11y-do">
        <h4>Do</h4>
        <p>Use Dark Grey (#222222) text on light backgrounds for maximum readability.</p>
      </div>
      <div className="a11y-dont">
        <h4>Don't</h4>
        <p>Use light gray text on white backgrounds or rely solely on color to convey meaning.</p>
      </div>
    </div>
  </div>

  <div className="a11y-section">
    <h2>Keyboard Navigation</h2>
    <p>
      All interactive elements must be accessible via keyboard. Users should be able to navigate, activate, and dismiss elements without a mouse.
    </p>

    <table className="a11y-table">
      <thead>
        <tr>
          <th>Key</th>
          <th>Action</th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <td><code>Tab</code></td>
          <td>Move focus to next interactive element</td>
        </tr>
        <tr>
          <td><code>Shift + Tab</code></td>
          <td>Move focus to previous interactive element</td>
        </tr>
        <tr>
          <td><code>Enter</code> / <code>Space</code></td>
          <td>Activate buttons, links, and controls</td>
        </tr>
        <tr>
          <td><code>Escape</code></td>
          <td>Close modals, dropdowns, and overlays</td>
        </tr>
        <tr>
          <td><code>Arrow Keys</code></td>
          <td>Navigate within menus, tabs, and radio groups</td>
        </tr>
      </tbody>
    </table>

    <h3>Focus Indicators</h3>
    <p>
      All focusable elements display a visible focus ring. Never remove focus outlines without providing an alternative indicator.
    </p>

  </div>

  <div className="a11y-section">
    <h2>ARIA Guidelines</h2>
    <p>
      Use ARIA attributes to enhance accessibility when semantic HTML alone isn't sufficient.
    </p>

    <h3>Common Patterns</h3>
    
    <table className="a11y-table">
      <thead>
        <tr>
          <th>Component</th>
          <th>ARIA Attributes</th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <td>Button (loading)</td>
          <td><code>aria-busy="true"</code>, <code>aria-disabled="true"</code></td>
        </tr>
        <tr>
          <td>Modal</td>
          <td><code>role="dialog"</code>, <code>aria-modal="true"</code>, <code>aria-labelledby</code></td>
        </tr>
        <tr>
          <td>Toast</td>
          <td><code>role="alert"</code>, <code>aria-live="polite"</code></td>
        </tr>
        <tr>
          <td>Dropdown Menu</td>
          <td><code>aria-expanded</code>, <code>aria-haspopup="true"</code></td>
        </tr>
        <tr>
          <td>Form Field</td>
          <td><code>aria-invalid</code>, <code>aria-describedby</code> (for errors)</td>
        </tr>
      </tbody>
    </table>

    <div className="a11y-do-dont">
      <div className="a11y-do">
        <h4>Do</h4>
        <p>Use semantic HTML first (button, nav, main, etc.), then enhance with ARIA only when needed.</p>
      </div>
      <div className="a11y-dont">
        <h4>Don't</h4>
        <p>Use ARIA to override or duplicate native semantics. Avoid <code>role="button"</code> on actual buttons.</p>
      </div>
    </div>
  </div>

  <div className="a11y-section">
    <h2>Images and Media</h2>
    
    <h3>Alt Text</h3>
    <p>
      All images must have appropriate alt text. The content depends on the image's purpose.
    </p>

    <table className="a11y-table">
      <thead>
        <tr>
          <th>Image Type</th>
          <th>Alt Text Approach</th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <td>Vehicle photos</td>
          <td>Describe the vehicle: "2025 Chevrolet Trax front three-quarter view"</td>
        </tr>
        <tr>
          <td>Decorative images</td>
          <td>Use empty alt: <code>alt=""</code></td>
        </tr>
        <tr>
          <td>Icons with text</td>
          <td>Use empty alt or <code>aria-hidden="true"</code></td>
        </tr>
        <tr>
          <td>Icon-only buttons</td>
          <td>Add <code>aria-label</code> to button: <code>aria-label="Close"</code></td>
        </tr>
      </tbody>
    </table>
  </div>

  <div className="a11y-section">
    <h2>Forms</h2>
    
    <ul className="a11y-checklist">
      <li>Every input has an associated label (visible or <code>aria-label</code>)</li>
      <li>Required fields are indicated visually and programmatically</li>
      <li>Error messages are associated with inputs via <code>aria-describedby</code></li>
      <li>Form validation doesn't rely solely on color</li>
      <li>Focus moves to first error on form submission failure</li>
      <li>Success/error states are announced to screen readers</li>
    </ul>

    <pre className="a11y-code">{`<TextField
  label="Email"
  type="email"
  required
  error="Please enter a valid email"
  aria-describedby="email-error"
/>`}</pre>
  </div>

  <div className="a11y-section">
    <h2>Testing Checklist</h2>
    <p>
      Use this checklist when building new components or reviewing existing ones.
    </p>
    
    <ul className="a11y-checklist">
      <li>Can all functionality be accessed with keyboard only?</li>
      <li>Is focus order logical and visible?</li>
      <li>Do all images have appropriate alt text?</li>
      <li>Is color contrast at least 4.5:1 for text?</li>
      <li>Are form fields properly labeled?</li>
      <li>Do error messages provide clear guidance?</li>
      <li>Can content be zoomed to 200% without loss of functionality?</li>
      <li>Does the component work with screen readers?</li>
      <li>Are animations respectful of reduced motion preferences?</li>
    </ul>
  </div>
